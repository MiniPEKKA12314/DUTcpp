## 什么是拷贝构造函数？

**拷贝构造函数**是一种特殊构造函数，用于根据一个已有对象创建一个**内容相同的新对象**。

典型声明：

```cpp
class MyClass
{
public:
    MyClass(const MyClass& other);
};
```

普通构造函数通常根据单独的值创建对象；拷贝构造函数则使用**同类型的另一个对象**初始化新对象。

---

# 为什么需要拷贝构造函数？

假设有一个简单类：

```cpp
class Student
{
private:
    string name;

public:
    Student(const string& n) : name(n) {}
};
```

创建一个对象：

```cpp
Student s1("Alice");
```

可以根据它创建一个拥有完全相同信息的新对象：

```cpp
Student s2(s1);
```

拷贝构造函数会将 `s1` 的状态复制到新创建的 `s2`。

```text
s1                 s2
+---------+       +---------+
| Alice   |       | Alice   |
+---------+       +---------+
```

注意：`s1` 和 `s2` 是**两个不同对象**，只是它们保存的数据相同。

---

# 默认拷贝构造函数

如果没有自己编写拷贝构造函数，编译器会自动生成一个：

```cpp
class Student
{
private:
    string name;
    int age;
};
```

编译器生成的版本会进行**逐成员复制**：

```cpp
Student s1("Alice", 20);
Student s2 = s1;
```

概念上类似：

```cpp
Student(const Student& other)
{
    name = other.name;
    age = other.age;
}
```

即每个数据成员依次复制。对于 `string`、`int` 等能安全复制的成员，这通常没有问题。

---

# 什么时候调用拷贝构造函数？

只要有一个**新对象**根据已有对象初始化，就可能调用拷贝构造函数。

## 情况 1：直接初始化

```cpp
Student s2(s1);
```

## 情况 2：拷贝初始化

```cpp
Student s2 = s1;
```

虽然看起来有 `=`，但 `s2` 正在创建，因此这仍是拷贝构造。

## 情况 3：按值传参

```cpp
void print(Student s)
{
}

print(s1);
```

参数 `s` 是一个新对象，通常由 `s1` 拷贝而来。

## 情况 4：按值返回

```cpp
Student createStudent()
{
    return Student("Alice");
}
```

按值返回对象可能调用拷贝构造函数；现代 C++ 常会进行拷贝消除（copy elision），从而省去这次复制。

---

# 拷贝构造函数与赋值运算符

这两个操作经常被混淆。

## 拷贝构造函数

创建一个**新对象**：

```cpp
Student s2(s1);
// 或 Student s2 = s1;
```

## 拷贝赋值运算符

将数据复制到一个**已经存在的对象**：

```cpp
Student s1("Alice");
Student s2("Bob");
s2 = s1;
```

`s2` 已存在，只是它的内容被改为与 `s1` 相同。

---

# 默认拷贝构造函数的问题

默认拷贝对普通数据成员表现良好，但拥有动态内存的类会出问题：

```cpp
class Buffer
{
private:
    int* data;

public:
    Buffer()
    {
        data = new int(10);
    }

    ~Buffer()
    {
        delete data;
    }
};
```

使用默认拷贝构造：

```cpp
Buffer b1;
Buffer b2 = b1;
```

默认的逐成员复制只会复制指针地址，因此 `b1.data` 和 `b2.data` 指向同一块内存：

```text
b1.data ----+
            v
          +----+
          | 10 |
          +----+
            ^
b2.data ----+
```

两个对象析构时都会执行：

```cpp
delete data;
```

同一块内存被释放两次，会导致**未定义行为**。

---

# 深拷贝

解决方法是自己定义拷贝构造函数，重新申请独立内存：

```cpp
class Buffer
{
private:
    int* data;

public:
    Buffer()
    {
        data = new int(10);
    }

    Buffer(const Buffer& other)
    {
        data = new int(*other.data);
    }

    ~Buffer()
    {
        delete data;
    }
};
```

现在：

```cpp
Buffer b2 = b1;
```

会让两个对象分别拥有独立内存：

```text
b1.data -> [10]
b2.data -> [10]
```

这叫**深拷贝**。每个对象管理自己的资源，析构时不会相互影响。

---

# 三法则（Rule of Three）

如果一个类自己管理资源，例如动态内存、文件句柄或 socket，通常要一起定义以下三个成员：

```text
析构函数
拷贝构造函数
拷贝赋值运算符
```

它们共同保证资源的所有权和复制行为正确。

现代 C++ 在此基础上加入移动构造函数和移动赋值运算符，称为**五法则（Rule of Five）**。更推荐优先使用 `string`、`vector`、`unique_ptr` 等 RAII 类型，尽量避免手写 `new` 和 `delete`。

---

# 重点总结

- 拷贝构造函数根据已有对象创建新对象。
- 典型声明为：

```cpp
MyClass(const MyClass& other);
```

- 编译器会自动生成默认拷贝构造函数，执行逐成员复制。
- 它常在以下场景调用：对象初始化、按值传参、按值返回。
- 拷贝构造创建新对象；拷贝赋值修改已有对象。
- 类拥有资源时通常需要深拷贝，避免多个对象共享同一资源。
- 管理资源的类通常遵循三法则或五法则。
