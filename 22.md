# C++ STL 容器入门

**C++；初学者路线；STL**

STL 是 C++ 自带的一组成熟工具，用于存储和组织数据。无需自己从零实现链表或哈希表，只需包含相应头文件即可使用。

# 什么是 STL？

STL（Standard Template Library，标准模板库）由三部分组成：

- **容器（Containers）**：保存数据，例如 `vector`、`map`。
- **迭代器（Iterators）**：类似指针，用于在容器中移动和定位元素。
- **算法（Algorithms）**：可复用操作，例如 `sort`、`find`，通过迭代器操作容器。

本文重点介绍第一部分：**容器**。

为什么不只用普通数组？

```cpp
int arr[10];
```

普通数组大小固定，不能方便地增长、缩小，也不直接知道自己有多少元素。STL 容器提供更灵活的接口和常用成员函数。

---

# 三类主要容器

| 家族 | 成员 | 排序/组织方式 |
|---|---|---|
| 顺序容器 | `vector`、`deque`、`list`、`array`、`forward_list` | 按插入位置保存顺序 |
| 关联容器 | `set`、`map`、`multiset`、`multimap` | 按键值排序 |
| 无序容器 | `unordered_set`、`unordered_map` | 按哈希桶存储，不保证顺序，但查找快 |
| 容器适配器 | `stack`、`queue`、`priority_queue` | 限制底层容器的访问方式 |

# `std::vector`：动态数组

**定义：**可自动增长、支持按下标快速访问的数组。

```cpp
#include <iostream>
#include <vector>

int main()
{
    std::vector<int> scores;
    scores.push_back(90);
    scores.push_back(85);
    scores.push_back(100);

    scores[1] = 88;

    for (int score : scores) {
        std::cout << score << " ";
    }

    std::cout << "\nsize = " << scores.size();
}
```

输出：

```text
90 88 100
size = 3
```

适用场景：大多数时候都应优先考虑 `vector`，特别是主要在尾部添加元素、并需要快速按下标访问时。

---

# `std::deque`：双端队列

**定义：**和 `vector` 类似，但在头部和尾部插入、删除都很快。

```cpp
#include <deque>
#include <iostream>
#include <string>

int main()
{
    std::deque<std::string> line;
    line.push_back("Alice");
    line.push_back("Bob");
    line.push_front("Zoe");

    for (const std::string& name : line) {
        std::cout << name << " ";
    }
}
```

输出：

```text
Zoe Alice Bob
```

适用场景：经常在头部增加或删除元素时。`vector` 在头部操作通常需要移动后续元素，效率较低。

---

# `std::list`：双向链表

**定义：**每个元素连接前后邻居。已知位置时，在任意处插入和删除较快，但不支持下标随机访问。

```cpp
#include <iostream>
#include <list>

int main()
{
    std::list<int> train;
    train.push_back(2);
    train.push_back(3);
    train.push_front(1);

    for (int car : train) {
        std::cout << car << "-";
    }
}
```

输出：

```text
1-2-3-
```

不能写：

```cpp
// train[1];  // list 不支持
```

必须使用迭代器或范围 `for` 循环逐个访问。

---

# `std::set`：有序且不重复的集合

**定义：**保存唯一元素，并自动按从小到大排序。

```cpp
#include <iostream>
#include <set>

int main()
{
    std::set<int> ids;
    ids.insert(5);
    ids.insert(1);
    ids.insert(5);  // 重复元素会被忽略
    ids.insert(3);

    for (int id : ids) {
        std::cout << id << " ";
    }
}
```

输出：

```text
1 3 5
```

适用场景：需要去重，并希望元素自动有序。

---

# `std::map`：有序键值对

**定义：**保存“键 -> 值”对，并按键排序。可将它理解为字典。

```cpp
#include <iostream>
#include <map>
#include <string>

int main()
{
    std::map<std::string, int> age;
    age["Alice"] = 22;
    age["Bob"] = 19;

    std::cout << "Bob is " << age["Bob"] << "\n";

    for (const auto& [name, years] : age) {
        std::cout << name << ": " << years << "\n";
    }
}
```

适用场景：想用姓名、ID、单词等有意义的键查找值，而不是只用数字下标。

---

# `std::unordered_map`：无序哈希表

**定义：**同样保存“键 -> 值”，但使用哈希表实现；平均查找速度更快，但遍历时不保证键的顺序。

```cpp
#include <iostream>
#include <string>
#include <unordered_map>

int main()
{
    std::unordered_map<std::string, int> inventory;
    inventory["apples"] = 10;
    inventory["bananas"] = 4;

    std::cout << inventory["apples"];
}
```

### `map` 和 `unordered_map` 的选择

- 需要键始终有序、遍历顺序稳定：使用 `map`。
- 只追求快速查找、不关心顺序：使用 `unordered_map`。

# 容器适配器

适配器通常以其他容器为底层存储，但限制访问方式以实现特定行为。

## `std::stack`：后进先出（LIFO）

```cpp
#include <iostream>
#include <stack>

std::stack<int> plates;
plates.push(1);
plates.push(2);
plates.push(3);
plates.pop();
std::cout << plates.top();  // 2
```

像一叠盘子，只能从顶部添加或取走。

## `std::queue`：先进先出（FIFO）

```cpp
#include <iostream>
#include <queue>
#include <string>

std::queue<std::string> line;
line.push("Alice");
line.push("Bob");
line.pop();
std::cout << line.front();  // Bob
```

像排队：最早到的人最先离开。

## `std::priority_queue`：优先级最高者在前

```cpp
#include <iostream>
#include <queue>

std::priority_queue<int> pq;
pq.push(3);
pq.push(9);
pq.push(1);
std::cout << pq.top();  // 9
```

默认情况下，最大元素总在顶部。

# 其他常见容器

| 容器 | 简述 |
|---|---|
| `array` | 固定长度数组，提供 STL 风格成员函数，大小不可改变。 |
| `forward_list` | 单向链表，比 `list` 更轻量，只能单向遍历。 |
| `multiset` | 类似 `set`，但允许重复值。 |
| `multimap` | 类似 `map`，但允许重复键。 |
| `unordered_set` | 类似 `set`，但哈希存储、无序、平均查找更快。 |

---

# 时间复杂度速查

- `O(1)`：元素数量增加时，操作时间基本不变。
- `O(log n)`：即使数据很多也较快。
- `O(n)`：可能需要依次检查所有元素。

| 容器 | 按下标访问 | 尾部插入/删除 | 搜索 |
|---|---|---|---|
| `vector` | `O(1)` | 平均 `O(1)` | `O(n)` |
| `deque` | `O(1)` | 两端 `O(1)` | `O(n)` |
| `list` | `O(n)` | 已知迭代器位置时 `O(1)` | `O(n)` |
| `set` / `map` | 不支持下标 | `O(log n)` | `O(log n)` |
| `unordered_map` | 不支持下标 | 平均 `O(1)` | 平均 `O(1)` |

`vector` 扩容时偶尔需要重新分配和复制元素，因此单次 `push_back` 可能是 `O(n)`，但平均复杂度仍为 `O(1)`。
