# C++ 模板与泛型编程入门

**C++；初学者路线；泛型编程**

模板让一段代码可以适用于**任意类型**，它也是整个 STL 的基础。

# 什么是模板？

**模板（template）**是函数或类的蓝图，其中类型先用占位符表示；实际使用时，编译器再根据传入的类型生成对应代码。这叫**泛型编程**。

你其实已经使用过模板：

```cpp
std::vector<int> scores;
```

`vector` 本身不知道也不关心保存的是何种类型；`<int>` 由你提供，告诉它元素类型。STL 的许多工具都建立在模板之上。

---

# 模板解决什么问题？

没有模板时，若想实现不同类型的最大值比较，可能需要重复编写逻辑：

```cpp
int maxInt(int a, int b)
{
    return (a > b) ? a : b;
}

double maxDouble(double a, double b)
{
    return (a > b) ? a : b;
}

string maxString(string a, string b)
{
    return (a > b) ? a : b;
}
```

这些函数逻辑完全相同。复制代码会增加维护成本：修复一个版本时，其他版本也可能需要同步修改。模板让这段逻辑只写一次。

# 函数模板

**定义：**类型作为参数的函数。`template<typename T>` 写在函数前，`T` 是类型占位符。

```cpp
#include <iostream>
#include <string>

template <typename T>
T myMax(T a, T b)
{
    return (a > b) ? a : b;
}

int main()
{
    std::cout << myMax(3, 7) << "\n";  // T = int
    std::cout << myMax(2.5, 1.1) << "\n";  // T = double
    std::cout << myMax(std::string("cat"), std::string("dog"));
}
```

输出：

```text
7
2.5
dog
```

### 模板参数推导

调用时通常不必写：

```cpp
myMax<int>(3, 7);
```

编译器会根据参数自动推断：

```cpp
myMax(3, 7);  // 自动推断 T 为 int
```

这叫**模板参数推导**。

---

# 多个类型参数

模板不限于一个类型占位符，可以用逗号分隔多个参数：

```cpp
#include <iostream>

template <typename T1, typename T2>
void printPair(T1 first, T2 second)
{
    std::cout << "(" << first << ", " << second << ")";
}

int main()
{
    printPair(1, "apple");
}
```

输出：

```text
(1, apple)
```

这里 `T1` 是 `int`，`T2` 是字符串字面量对应的类型。

# 类模板

**定义：**数据成员类型由模板参数决定的类。创建对象时通常需要明确写出类型。

```cpp
#include <iostream>
#include <string>

template <typename T>
class Box
{
private:
    T value;

public:
    Box(T input) : value(input) {}

    T get() const
    {
        return value;
    }
};

int main()
{
    Box<int> intBox(42);
    Box<std::string> stringBox("hello");

    std::cout << intBox.get() << " " << stringBox.get();
}
```

输出：

```text
42 hello
```

这与 `std::vector<T>` 的思路相同：`T` 代表容器中元素的类型。

# 非类型模板参数

模板参数也可以是一个值，而不是类型。常用于编译期固定大小：

```cpp
#include <iostream>

template <typename T, int Size>
class FixedArray
{
private:
    T data[Size];

public:
    int length() const
    {
        return Size;
    }
};

int main()
{
    FixedArray<double, 10> array;
    std::cout << "Length: " << array.length();
}
```

输出：

```text
Length: 10
```

`Size` 是编译期常量，大小也成为类型的一部分。这与 `std::array<T, N>` 的设计类似。

# 其他模板概念

| 概念 | 简述 |
|---|---|
| 默认模板参数 | `template<typename T = int>`：存在合理默认类型时可省略类型。 |
| 模板特化 | 为某一种特定类型提供特殊实现，例如单独处理 `bool`。 |
| 可变参数模板 | `template<typename... Args>`：接受任意数量的类型参数。 |
| `auto` 与模板 | C++20 中 `void print(auto value)` 可看作单参数函数模板的简写。 |

---

# 常见注意点

模板按实际使用的类型生成代码。例如：

```cpp
myMax(3, 7);      // 编译器生成 int 版本
myMax(2.5, 1.1);  // 编译器生成 double 版本
```

背后会生成两个不同函数版本。由于编译器在使用模板时必须看到完整定义，模板的定义通常放在头文件中，而不能只把声明放进头文件、定义放进普通 `.cpp` 文件。

模板内部使用的操作必须对 `T` 有效。例如 `myMax` 使用了 `>`，因此传入的类型必须支持比较运算符 `>`。

# 语法速查

| 目标 | 语法 |
|---|---|
| 泛型函数 | `template<typename T> T f(T x) { ... }` |
| 多类型参数 | `template<typename T1, typename T2>` |
| 泛型类 | `template<typename T> class Box { ... };` |
| 值作为参数 | `template<typename T, int N>` |
| 使用类模板 | `Box<int> box(5);` |

# 练习建议

1. 编写函数模板：

```cpp
template <typename T>
void swapValues(T& a, T& b)
```

它可以交换任意类型的两个变量。分别使用两个 `int` 和两个 `string` 测试。

2. 编写 `Pair<T>` 类模板，存储两个相同类型的值，并提供 `sum()` 成员函数。先用数字类型测试；注意只有支持 `+` 的 `T` 才能使用该函数。
